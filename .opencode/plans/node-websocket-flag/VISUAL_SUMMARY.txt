┌─────────────────────────────────────────────────────────────────────────────┐
│                            KEY DESIGN DECISIONS                              │
└─────────────────────────────────────────────────────────────────────────────┘

1. SCHEMA-LEVEL FLAG (Not Instance-Level)
   ✅ Capability is per-type, not per-instance
   ✅ Simpler: No database, no user toggles
   ✅ Performant: Loaded once at startup

2. DEFAULT TO TRUE (Not False)
   ✅ Backward compatible (existing nodes unaffected)
   ✅ Safe fallback (unknown types show controls)
   ✅ Fail-safe (better to show than hide incorrectly)

3. UI-ONLY ENFORCEMENT (Not DAG-Level)
   ✅ Separation of concerns (UI ≠ orchestrator logic)
   ✅ Natural enforcement (nodes without outputs don't stream anyway)
   ✅ Flexibility (future streaming types possible)

4. SINGLE BOOLEAN (Not Separate Flags)
   ✅ YAGNI: One flag sufficient for current use cases
   ✅ Simpler: Less to maintain, fewer test combinations
   ✅ Extensible: Can add granular flags later if needed

5. ANGULAR SIGNALS (Not RxJS)
   ✅ Modern Angular pattern (v16+)
   ✅ Automatic dependency tracking
   ✅ Better template optimization

6. TEMPLATE GUARDS (@if, Not CSS)
   ✅ Performance: Elements removed from DOM
   ✅ Accessibility: Screen readers don't see hidden controls
   ✅ Simpler: No CSS class management
